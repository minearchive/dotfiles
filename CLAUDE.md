# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A NixOS + Home Manager dotfiles repository managing multiple hosts and user profiles with Nix Flakes. Wayland-first desktop environment built around Hyprland, QuickShell (QML), and Material Design 3 theming via Matugen.

## Applying Configurations

**System (NixOS):**
```bash
sudo nixos-rebuild switch --flake .#msiLaptop   # Intel laptop
sudo nixos-rebuild switch --flake .#desktop     # NVIDIA desktop
sudo nixos-rebuild switch --flake .#wsl         # WSL
```

**Home Manager:**
```bash
home-manager switch --flake .#material   # Full desktop profile
home-manager switch --flake .#pastel     # Alternative theme profile
home-manager switch --flake .#wsl        # WSL minimal profile
```

**Convenience scripts (WSL-focused):**
```bash
./rebuild.sh       # sudo nixos-rebuild switch --flake .#wsl
./update-home.sh   # home-manager switch --flake .#wsl -b backup
```

## Architecture

### Flake Outputs

- **nixosConfigurations**: `msiLaptop`, `desktop`, `wsl`
- **homeConfigurations**: `material`, `pastel`, `wsl`

### System Config (`hosts/`)

- `hosts/shared/configuration.nix` ŌĆö shared base: locale (ja_JP), fcitx5-mozc IME, PipeWire audio, Tailscale, xremap (CapsLockŌåÆCtrl), Hyprland+UWSM, GDM
- `hosts/msiLaptop/` ŌĆö Intel GPU drivers, KDE Connect, i915 kernel params
- `hosts/desktop/` ŌĆö NVIDIA open drivers, Lanzaboote secure boot, nvidia-container-toolkit, Samba
- `hosts/wsl/` ŌĆö minimal WSL config

`hardware-configuration.nix` files are auto-generated by `nixos-generate-config` ŌĆö don't edit manually.

### Home Manager (`home.nix`, `material/`, `pastel/`, `wsl/`)

- **material** ŌĆö primary desktop profile; imports `home.nix` ŌåÆ `material/material.nix`
  - `material/development.nix` ŌĆö dev tools: gcc, go, python312, nodejs_22, bun, deno, jdk, nixd, devenv, direnv, IntelliJ IDEA
  - `material/quickshell.nix` ŌĆö copies QML files to `~/.local/share/qs-bar/`, triggers Matugen, launches QuickShell via activation hook
  - `material/programs/hyprland.nix` ŌĆö full Hyprland config (keybinds, animations, startup, gestures)
  - `material/programs/` ŌĆö per-app configs (kitty, neovim, firefox, zen, git, mako, wlogout, spotify, etc.)
  - `material/templates.nix` ŌĆö Nix dev templates (large file)
- **pastel** ŌĆö alternative theming (Bibata cursors), different app selection
- **wsl** ŌĆö fish shell + claude-code, no Wayland

### QuickShell UI (`material/quickshell/`)

QML-based desktop shell components. Changes here require re-running the activation hook (or `home-manager switch`) to copy files to `~/.local/share/qs-bar/` and restart the quickshell process. Key structure:
- `shell.qml` ŌĆö root
- `topbar/` ŌĆö panel components (clock, media player, hardware monitors, notifications, workspace switcher)
- `theme/` ŌĆö `Colors.qml`, `Fonts.qml` (populated by Matugen)
- `appLauncher/`, `popout/`, `service/`, `component/`, `shader/`

### Theming

Matugen generates Material Design 3 colors from the current wallpaper. Colors flow into QuickShell QML theme files and Qt5ct config. The wallpaper script (`~/.config/hypr/script/wallpaper_change.sh`) triggers Matugen on wallpaper change.
